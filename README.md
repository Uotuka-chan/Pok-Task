# PokéTask

Webアプリケーション開発実習の最終成果物です。SPEC駆動開発の考え方に基づき、実装範囲を小さく定義し、DB・API・UIが一貫してつながることを重視しました。

## 概要

PokéTaskは、ポケモン対戦における育成論を管理するためのWebアプリケーションです。
対戦用ポケモンの性格、努力値、持ち物、技構成、育成メモを登録し、育成状況を一覧で管理できます。
育成途中のポケモンや完成済みの育成論を可視化することで、複数の育成計画を効率よく管理できるよう設計しました。
また、ポケモン名に対応した画像を自動表示する機能を実装しており、視覚的にも管理しやすいUIを目指しました。

## 開発目的

育成論では、
* 育成予定のポケモン
* 努力値配分
* 技構成
* 持ち物
* 育成状況

などを複数同時に管理する必要があります。
しかし、メモ帳やスクリーンショットでは管理が煩雑になりやすく、育成途中の個体を把握しづらいという課題があります。
そこで本アプリでは、育成論をデータベースで管理し、一覧表示できる環境を構築することを目的としました。

## 主な機能

### 1. 育成論登録機能

以下の情報を登録できます。

* ポケモン名
* 性格
* 持ち物
* 努力値
* 技
* メモ

### 2. 育成状況管理機能

登録した育成論に対して
* 未完了
* 完了
を切り替えることができます。
育成済み個体の管理が容易になります。

### 3. 検索機能

ポケモン名による検索機能を実装しています。
登録数が増えても目的の育成論を素早く発見できます。

### 4. ポケモン画像自動表示機能

ポケモン名に対応する画像を自動表示します。
画像URLを入力しなくても視覚的に判別できるよう工夫しました。

## 工夫した点

### コンポーネント分割

以下のように責務を分離しました。

* TaskCard
* TaskList
* StateMessage

再利用しやすく保守性の高い構造を意識しました。

### リアルタイム計算

努力値を入力した瞬間に合計値を更新し、入力ミスを防止できるようにしました。

### 視認性の向上

ポケモン画像を表示することで、文字だけではなく視覚的に育成論を管理できるよう工夫しました。

### GitHubによるバージョン管理

GitHubを利用してソースコードを管理し、変更履歴を記録できる環境を構築しました。

## フリー記述レポート

本アプリでは、授業で学んだSPEC駆動開発の考え方を意識しながら開発を行った。まず「どのような課題を解決するのか」を整理し、ポケモン対戦における育成論管理というテーマを設定した。その上で必要な機能を洗い出し、画面設計、データ設計、API設計を段階的に進めた。

実装ではNext.js、TypeScript、Prisma、SQLiteを使用し、フロントエンドからデータベースまで一貫した構成を構築した。特にコンポーネント分割を意識し、TaskCard、TaskList、StateMessageなど責務ごとに分離することで保守性を高めた。

また、努力値のリアルタイム集計機能やポケモン画像の自動表示機能など、利用者が直感的に使いやすいUIを意識して実装した。さらにGitHubによるソースコード管理を行い、変更履歴を追跡できる開発環境を構築した。

今後はダメージ計算機能やポケモンAPIとの連携機能を追加し、実際の対戦準備にも活用できるアプリへ発展させたいと考えている。
